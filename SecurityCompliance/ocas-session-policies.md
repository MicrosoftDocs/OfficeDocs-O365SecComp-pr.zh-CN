# <a name="session-policies-in-office-365-cloud-app-security"></a>Office 365 中的会话策略云应用安全

|:-----|:-----|:-----|:-----|

office 365 云应用安全会话策略启用实时会话级别监控, affording 您可以精细查看 Office 365 应用程序, 以及根据您为用户会话设置的策略执行不同操作的能力。通过会话控制, 您可以在监视会话和/或限制特定会话活动时使用条件访问应用程序控件的反向代理功能, 而无需完全允许或阻止访问。

例如, 您可以确定来自非托管设备或来自特定位置的会话, 您希望允许用户访问应用程序, 但也限制下载敏感文件或要求在下载时保护特定文档。通过会话策略, 您可以设置这些用户会话控件并允许访问, 并使您能够:

- [监视所有活动](#monitor-all-activities)

- [阻止所有下载](#block-all-downloads)

- [阻止特定活动](#block-specific-activities)

- [在下载时保护文件](#protect-files-on-download)

## <a name="prerequisites-to-using-session-policies"></a>使用会话策略的先决条件

- Azure AD 高级 P1 许可证

- 应 [使用条件访问应用程序控件部署相关的](ocas-deploy-conditional-access-app-control.md)Office 365 应用程序

- 应就地将用户重定向到 Office 365 云应用安全性的 [Azure AD 条件访问策略](https://docs.microsoft.com/azure/active-directory/active-directory-conditional-access-azure-portal) 

## <a name="create-an-azure-ad-conditional-access-policy"></a>创建 Azure AD 条件访问策略

Azure Active Directory 条件访问策略和云应用安全会话策略一起运行, 以检查每个用户会话并为每个应用制定策略决策。若要在 Azure AD 中设置条件访问策略, 请按照以下过程操作:

1. 使用 "条件访问应用程序" 控件的一个用户或一组用户以及要控制的应用程序的分配来配置 [Azure AD 条件访问策略](https://docs.microsoft.com/azure/active-directory/active-directory-conditional-access-azure-portal) 。注意: 仅此策略会影响 [使用条件访问应用程序控件](ocas-deploy-conditional-access-app-control.md) 部署的应用程序。

2. 通过在 " **会话** " 页中选择 " **使用条件访问应用程序控制强制性限制** " 将用户路由到 Office 365 云应用安全性。

## <a name="create-a-cloud-app-security-session-policy"></a>创建云应用安全会话策略

若要创建新的会话策略, 请按照以下过程操作:

1. 在门户中, 依次选择 " **控制** " 和 " **策略**"。

2. 在 " **策略** " 页中, 单击 " **创建策略** ", 然后选择 " **会话策略**"。

3. 在 " **会话策略** " 窗口中, 为策略分配名称

4. 在 " **会话控制类型** " 字段中:
    
    - 仅当您只想监视用户的活动时, **才** 选择 "监视器"。此选项将为您选择的应用程序创建 "仅监视" 策略, 在这些应用程序中, 将下载所有登录、启发式下载和活动类型。
    
    - 如果要监视用户活动, 请选择 " **控制文件下载 (使用 DLP)** "。您可以执行其他操作, 如阻止或保护用户的下载。
    
    - 选择 " **阻止活动** " 以阻止特定活动, 您可以使用 **活动类型** 筛选器选择这些活动。将监视 (并在活动日志中报告) 所选应用中的所有活动。如果选择 " **阻止** " 操作, 则将阻止您选择的特定活动。如果选择 " **测试** 操作" 并启用 "通知", 则您选择的特定活动将引发警报。

5. 在 " **活动源** " 下, 在 **匹配以下** 所有部分的活动中, 选择要应用于策略的其他活动筛选器。这些筛选器可以包含以下选项:
    
    - **设备标记**: 使用此筛选器可标识非托管设备。
    
    - **位置**: 使用此筛选器可确定未知 (因而是有风险的) 位置。
    
    - **IP 地址**: 使用此筛选器筛选每个 IP 地址或使用之前分配的 ip 地址标记。
    
    - **用户代理标记**: 使用此筛选器可启用启发, 以确定移动和桌面应用。此筛选器可设置为等于或不等于 **本机客户端**。应针对每个云应用为你的移动和桌面应用测试此筛选器。<br>注意: 会话策略不支持移动和桌面应用。也可以通过创建访问策略来阻止或允许移动应用和桌面应用。

6. 如果选择了用于 **控制文件下载 (使用 DLP)** 的选项, 请在 " **活动源** " 下的 " **文件与以下** 所有内容匹配" 部分中, 选择 "其他文件筛选器" 以应用于该策略。这些筛选器可以包含以下选项:
        
    - **分类标签** -如果你的组织使用 Azure 信息保护, 并且你的数据已由其分类标签保护, 则使用此筛选器。您可以根据应用于这些文件的分类标签对文件进行筛选。有关与 azure 信息保护的集成的详细信息, 请参阅 [azure 信息保护集成](https://docs.microsoft.com/cloud-app-security/azip-integration)。
        
    - **** 文件名-使用此筛选器将策略应用于特定文件。
        
    - **文件类型** -使用此筛选器将策略应用于特定文件类型, 例如, 阻止下载所有 .xls 文件。
    
    - 在 " **内容检查** " 部分中, 设置是否要使 DLP 引擎能够扫描文档和文件内容。
    
    - 在 " **操作**" 下, 选择以下项之一:
        
        - **测试 (监视所有活动)**: 将此操作设置为根据您设置的策略筛选器明确允许下载。
        
        - **阻止 (阻止文件下载和监视所有活动)**: 将此操作设置为根据您设置的策略筛选器显式阻止下载。有关详细信息, 请参阅 [阻止下载的工作原理](https://docs.microsoft.com/en-us/cloud-app-security/session-policy-aad#block-download)。
        
        - **保护 (应用分类标签以下载和监视所有活动)**: 只有在 " **会话策略**" 下选择了 " **控制文件下载 (使用 DLP)** " 时, 此选项才可用。如果你的组织使用 azure 信息保护, 则可以将 **操作** 设置为对文件应用 azure 信息保护中的分类标签集。有关详细信息, 请参阅 [保护下载的工作原理](https://docs.microsoft.com/en-us/cloud-app-security/session-policy-aad#protect-download)。

7. 您 可以 **为具有策略严重性的每个匹配事件创建一个警报**, 并设置一个警报限制。选择要将警报作为电子邮件、短信还是同时作为这两者。

## <a name="monitor-all-activities"></a>监视所有活动

创建会话策略时, 匹配策略的每个用户会话将被重定向到会话控制, 而不是直接重定向到应用。用户将看到一个监控通知, 让他们知道他们的会话已被监视。

如果您不想通知用户他们被监视, 可以禁用通知消息。

1. 在 "设置" cog 下, 选择 " **常规设置**"。

2. 然后, 在 " **条件 Access 应用程序控制** " 下, 选择 " **用户监视** ", 然后取消选中 " **通知用户** " 复选

若要将用户保留在会话中, 条件访问应用程序控件将使用 Office 365 云应用安全 url 替换应用程序会话中的所有相关 url、Java 脚本和 cookie。例如, 如果应用返回的页面的链接的域以其所属的`myapp.com`域结束, 则条件 Access 应用程序控件将使用以类似`myapp.com.us.cas.ms`方式结尾的域替换链接。通过这种方式, 将通过 Office 365 云应用安全性监视整个会话。

条件访问应用程序控件记录通过它路由的每个用户会话的流量日志。流量日志包括时间、IP、用户代理、访问的 url 以及已上载和下载的字节数。将对这些日志进行分析, 并将连续报告 " **云应用安全条件访问应用程序" 控件**添加到云发现仪表板中的云发现报告列表中。

### <a name="to-export-these-logs"></a>若要导出这些日志:

1. 转到 "设置" "cog", 然后单击 " **条件访问应用程序控制**"。

2. 在表的右侧, 单击 "导出" 按钮。<br>!["导出" 按钮](media/image3.png)<br>

3. 选择报告的范围, 然后单击 " **导出**"。此过程可能需要一段时间。

### <a name="to-download-the-exported-log"></a>下载导出的日志:

1. 报告准备就绪后, 转到 " **设置** ", 然后按 " **导出报告**"。

2. 在表中, 从 **条件访问应用程序控制流量日志** 列表中选择相关报告, 然后单击 "下载"。<br>![下载按钮](media/image4.png)<br>

## <a name="block-all-downloads"></a>阻止所有下载

将 **Block** 设置为您希望在云应用安全会话策略中执行的 **操作** 时, 条件访问应用程序控件将阻止用户按照策略的文件筛选器下载文件。当用户开始下载时, 每个应用的 Office 365 云应用安全将识别下载事件。条件访问应用程序控制实时 intervenes 阻止其运行。当收到用户已启动下载的信号时, 条件 Access 应用程序控件将向用户返回 **下载限制** 的邮件, 并将下载的文件替换为文本文件。可以在会话策略中配置和自定义文本文件的邮件。

## <a name="block-specific-activities"></a>阻止特定活动

将 " **阻止活动** " 设置为 " **活动类型**" 时, 可以选择要在特定应用程序中阻止的特定活动。将在活动日志中监视和报告来自所选应用的所有活动。如果选择 " **阻止** " 操作, 则将阻止您选择的特定活动。如果选择 " **测试**操作" 并启用 "通知", 则您选择的特定活动将引发警报。

**阻止特定的活动** 并将其应用到特定组, 以为您的组织创建全面的只读模式。

## <a name="protect-files-on-download"></a>在下载时保护文件

选择 " **阻止活动** " 以阻止特定活动, 您可以使用 **活动类型** 筛选器找到这些活动。将监视 (并在活动日志中报告) 所选应用中的所有活动。如果选择 " **阻止** " 操作, 则将阻止您选择的特定活动。如果选择 " **测试** 操作" 并启用 "通知", 则您选择的特定活动将引发警报。

如果将 " **保护** " 设置为在云应用安全会话策略中执行的 **操作** , 则条件访问应用程序控件将根据策略的文件筛选器强制对文件进行标记和后续保护。标签在 Azure 信息保护控制台中配置, 并且必须在标签中选择 " **保护** " 以使其在云应用安全策略中显示为一个选项。如果选择了标签, 并且下载了符合云应用安全策略条件的文件, 则会在下载时对该文件应用标签和相应的保护 (带有权限)。在下载的文件现在受到保护时, 原始文件仍保持原样在云应用程序中。尝试访问该文件的用户必须满足应用的保护所确定的权限要求。
