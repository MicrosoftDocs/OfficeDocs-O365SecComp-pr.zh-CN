# <a name="message-trace-in-the-security--compliance-center"></a>安全 & 合规性中心中的邮件跟踪

邮件跟踪中安全 & 合规性中心跟随旅行通过 Exchange Online 组织的电子邮件。您可以确定一条消息，是否已收到、 拒绝、 延迟，或由服务发送。它还会显示其达到其最终状态之前，对邮件采取了哪些操作。

Exchange 管理员中心 (EAC) 中可用的邮件跟踪时提高了安全性 & 合规性中心中的邮件跟踪。邮件跟踪的信息可用于高效地用户有关其邮件发生了什么变化、 排查邮件流问题和解答验证策略更改。

## <a name="open-message-trace"></a>打开的邮件跟踪

1. 使用您的 工作或学校帐户[登录到 Office 365](https://support.office.com/article/e9eb7d51-5430-4929-91ab-6157c5a050b4)。

2. 在左上角选择应用启动器图标 ![Office 365 应用启动器图标](media/0aaa6945-f9a4-4b13-bf5f-d5c5dbe978fb.png)，然后选择" **管理**"。

3. 在左导航窗格中，展开**管理中心**，然后选择**安全 & 合规性**。

4. 在**安全 & 合规性**页打开中，展开**邮件流**，并选择**邮件跟踪**。

## <a name="message-trace-page"></a>邮件跟踪页面

从此处，您可以通过单击**启动跟踪**按钮开始新的默认跟踪。这将搜索所有邮件的所有发件人和收件人最近两天。或您可以使用的可用的查询类别中的存储查询之一，然后运行它们作为-是或将其用作启动您自己的查询的点：

- **默认查询**： Office 365 提供的内置查询。

- **自定义查询**： 查询保存由以供将来使用组织中的管理员。

- **自动保存查询**： 在最后一个十最近运行查询。此列表便于拿起您离开的地方。

此外在此页是**Downloadable 报告**部分请求已提交，以及他们自己的报告时可供下载。

## <a name="options-for-a-new-message-trace"></a>新的邮件跟踪的选项

### <a name="filter-by-senders-and-recipients"></a>按发件人和收件人筛选

默认值为**所有发件人**和**所有收件人**，但您可以使用以下字段来筛选结果：

- **通过这些人员**： 单击此字段以从您的组织选择一个或多个发件人。您还可以开始键入一个名称并将通过已键入的内容，得多类似的搜索页的行为方式筛选列表中的项。

- **向这些人**： 单击此字段以选择您的组织中的一个或多个收件人。

您还可以键入外部发件人和收件人的电子邮件地址。支持通配符 (`*@contoso.com`或`scot?@contoso.com`)，但不是能在同一时间的同一字段中使用多个通配符条目。

### <a name="more-search-options"></a>多个搜索选项

#### <a name="delivery-status"></a>传递状态

您可以将保留**所有**选择，默认值，或者您可以选择要筛选结果的下列值之一：

- **传递**： 邮件已成功递送到预定的目标。

- **挂起**： 正在邮件传递尝试或重新尝试。

- **Expanded**： 通讯组收件人已扩展之前传递到单个组的成员。

- **失败**： 消息未能送达。

- **隔离**： 邮件已隔离 （作为垃圾邮件、 批量邮件或网络钓鱼邮件）。有关详细信息，请参阅[Office 365 中的隔离电子邮件](https://support.office.com/article/4c234874-015e-4768-8495-98fcccfc639b.aspx)。

- **为垃圾邮件筛选**： 该消息标识垃圾邮件和已拒绝或阻止 （不隔离）。

- **获取状态：** Office 365 最近收到邮件，但没有其他状态数据尚不可用。签入几分钟后。

**注意**：**未决、** **隔离**，并**为垃圾邮件筛选器**的值仅适用于搜索少于 10 天。此外，可能有 5 至 10 分钟的延迟之间的实际和报告的传递状态。

#### <a name="find-related-records-for-this-message"></a>此消息的查找相关的记录

相关的消息记录所记录的共享相同的邮件 id。请记住，即使两人之间发送一条消息可以生成多个记录。记录数增加时邮件受通讯组扩展、 转接、 邮件流规则 （也称作传输规则） 等。

邮件跟踪详细信息包含不存在的摘要报告中的以下附加信息：

- **消息事件**： 此部分包含帮助分类邮件该服务采取的操作的分类。下面是一些您可能遇到的值得事件：

   - **接收**： 服务收到邮件。

   - **发送**： 由服务发送邮件。

   - **失败**： 消息未能送达。

   - **提供**： 邮件已传递到邮箱。

   - **展开**： 邮件发送给已扩展的通讯组。

   - **转接**： 由于内容转换、 邮件收件人限制或代理，收件人被移动到分叉的邮件。

   - **Defer**： 邮件传递延迟，可能会稍后重新尝试。

   - **已解决**： 邮件已被重定向到基于 Active Directory 查找新收件人地址。这种情况下，在原始收件人地址以及邮件的最终的传递状态邮件跟踪中单独行中列出。

   请注意，即使成功传递的一般的消息将在邮件跟踪生成多个**事件**条目。

- **详细信息**： 此部分包含以下详细信息：

   - **邮件 ID**： 此值在本主题前面的[邮件 ID](#message-id)部分所述。例如， `<d9683b4c-127b-413a-ae2e-fa7dfb32c69d@DM3NAM06BG401.Eop-nam06.prod.protection.outlook.com>`。

   - **邮件大小**

   - **从 IP**： 发送邮件的计算机的 IP 地址。从 Exchange Online 发送出站邮件，此值为空。

   - **为 IP**： 的 IP 地址或地址服务尝试传递邮件。如果消息有多个收件人，它们显示。发送到 Exchange Online 的入站邮件，此值为空。

### <a name="enhanced-summary-reports"></a>增强的摘要报告

在开始邮件跟踪**Downloadable 报告**部分提供了可用 （已完成） 的增强型摘要报告。是可用的报告中的以下信息：

- **origin_timestamp**<sup>*</sup>： 的日期和时间最初由使用配置的 UTC 时区的服务接收邮件。

- **sender_address 别名**： 发件人的电子邮件地址 (*别名*@*域*)。

- **Recipient_status**: 邮件的传递给收件人的状态。如果邮件发送给多个收件人，它将为每个格式显示所有收件人和相应的状态： \<*电子邮件地址*\>##\<*状态*\>。例如：

   - **##Receive，发送**表示服务已接收邮件，并被发送到预定的目标。

   - **##Receive，失败**，则意味着收到邮件服务，但传递给目标失败。

   - **##Receive，传递**表示服务已接收邮件，并已传递到收件人的邮箱。

- **message_subject**： 邮件的**主题**字段的前 256 个字符。

- **total_bytes**： 以字节为单位，包括附件在内的邮件的大小。

- **message_id**： 此值在本主题前面的[邮件 ID](#message-id)部分所述。例如， `<d9683b4c-127b-413a-ae2e-fa7dfb32c69d@DM3NAM06BG401.Eop-nam06.prod.protection.outlook.com>`。

- **network_message_id**： 可能由于分叉或通讯组扩展中创建的消息的所有副本仍然存在一个唯一的消息 ID 值。示例值是`1341ac7b13fb42ab4d4408cf7f55890f`。

- **original_client_ip**： 发件人客户端的 IP 地址。

- **方向性**： 指示是否发送邮件 （1） 向您的组织的入站或是否发送出站 （2） 从您的组织。

- **connector_id**： 源或目标连接器的名称。有关 Exchange Online 中的连接器的详细信息，请参阅[配置邮件流，使用 Office 365 中的连接器](https://docs.microsoft.com/Exchange/mail-flow-best-practices/use-connectors-to-configure-mail-flow/use-connectors-to-configure-mail-flow)。

- **delivery_priority**<sup>*</sup>： 是否具有**高**、**低**或**普通**优先级发送邮件。

<sup>*</sup>这些属性才可用增强型摘要报告中。

### <a name="extended-reports"></a>扩展的报告

邮件跟踪的开头**Downloadable 报告**部分提供了可用 （已完成） 扩展报告。（除外**origin_timestamp**和**delivery_priority**） 扩展报告中提供几乎所有增强型摘要报告的信息。仅扩展报告中提供的以下附加信息：

- **client_ip**： 提交邮件的邮件客户端的电子邮件服务器的 IP 地址。

- **client_hostname**： 的主机名或电子邮件服务器或提交邮件的邮件客户端的 FQDN。

- **server_ip**： 源或目标服务器的 IP 地址。

- **server_hostname**： 的主机名或目标服务器的 FQDN。

- **source_context**： 与**源**字段的额外信息。例如：

   - `Protocol Filter Agent`

   - `3489061114359050000`

- **源**： Exchange Online 的组件，负责该事件。例如：

   - `AGENT`

   - `MAILBOXRULE`

   - `SMTP`

- **event_id**： 这些对应于[查找此消息的相关的记录](#find-related-records-for-this-message)部分中所述的**消息事件**值。

- **internal_message_id**： 由当前正在处理邮件的 Exchange Online 服务器分配消息标识符。

- **recipient_address**： 邮件的收件人的电子邮件地址。由分号 （;） 分隔多个电子邮件地址。

- **recipient_count**： 的消息中的收件人总数。

- **related_recipient_address**： 用于`EXPAND`， `REDIRECT`，和`RESOLVE`事件显示其他收件人电子邮件地址与邮件相关联。

- **参考**： 此字段包含特定类型的事件的其他信息。例如：

   - **DSN**： 包含报告链接，这是**message_id**值的关联的传递状态通知 （也称为 DSN、 未送达报告、 NDR 或退回邮件），如果 DSN 生成后此事件。如果这是 DSN 邮件时，此字段包含的 DSN 生成的原始邮件**message_id**值。

   - **展开**： 包含**related_recipient_address**值相关的消息的数目。

   - **接收**： 可能包含相关的消息的**message_id**值，如果邮件生成的其他进程 （如收件箱规则）。

   - **发送**： 包含的任何 DSN 邮件**internal_message_id**值。

   - **转接**： 包含**internal_message_id**值的邮件的正在 （例如，通过内容转换、 邮件收件人限制或代理） 分为两路。

   - **MAILBOXRULE**： 包含导致生成以下出站消息的收件箱规则的入站邮件**internal_message_id**值。

   对于其他类型的事件，此字段通常为空。

- **return_path**： 通过发送邮件的**MAIL FROM**命令指定返回的电子邮件地址。尽管此字段也不为空，但也可以有 null 发件人地址值表示为`<>`。

- **message_info**： 邮件的其他信息。例如：

   - 消息发起日期-时间为 utc 格式`DELIVER`和`SEND`事件。发起日期-时间是邮件首次输入 Exchange Online 组织的时间。UTC 日期-时间表示 ISO 8601 日期时间格式： `yyyy-mm-ddThh:mm:ss.fffZ`，其中`yyyy`= 年`mm`= 月`dd`= 日，`T`指示的时间组件，开始`hh`= 小时， `mm` = 分钟， `ss` = 第二、 `fff`= 秒，和`Z`表示`Zulu`，这是另一种方法来表示 UTC。

   - 身份验证错误。例如，您可能会看到值`11a`和身份验证出错时使用的身份验证类型。

- **tenant_id**: GUID 值，该值代表 Exchange Online 组织 (例如， `39238e87-b5ab-4ef6-a559-af54c6b07b42`)。

- **original_server_ip**： 原始服务器的 IP 地址。

- **custom_data**： 包含与特定事件类型的数据。有关详细信息，请参阅以下各节。

#### <a name="customdata-values"></a>custom_data 值

**Custom_data**字段`AGENTINFO`各种 Exchange Online 代理使用事件日志消息处理的详细信息。以下各节介绍一些值得代理。

#### <a name="spam-filter-agent"></a>垃圾邮件筛选器代理

**Custom_data**值开头的`S:SFA`是垃圾邮件筛选器代理。下表中描述的关键详细信息：

|**值**|**说明**|
|:-----|:-----|
|`SFV=NSPM`|邮件被标记为非垃圾邮件并发送给预期收件人。|
|`SFV=SPM`|邮件由内容筛选器标记为垃圾邮件。|
|`SFV=BLK`|跳过筛选但阻止邮件，因为它是由已阻止发件人发送。|
|`SFV=SKS`|在内容筛选器处理之前，邮件被标记为垃圾邮件。这包括符合以下传输规则条件的邮件：自动将邮件标记为垃圾邮件并规避其他所有筛选。|
|`SCL=<number>`|有关不同的 SCL 值及其含义的详细信息，请参阅[垃圾邮件可信度](https://technet.microsoft.com/library/jj200686.aspx)。|
|`PCL=<number>`|邮件的仿冒可能性等级 (PCL) 值。可按照[垃圾邮件可信度](https://technet.microsoft.com/library/jj200686.aspx)中介绍 SCL 值的方式对这些值做出解释。  |
|`DI=SB`|已阻止邮件发件人。|
|`DI=SQ`|邮件已隔离。|
|`DI=SD`|邮件已删除。|
|`DI=SJ`|邮件已发送至收件人的"垃圾邮件"文件夹。|
|`DI=SN`|邮件已通过高风险传送池路由。有关详细信息，请参阅[高风险传递池用于出站邮件](https://technet.microsoft.com/library/jj200746.aspx)。|
|`DI=SO`|邮件已通过正常出站传送池路由。|
|SFS = [a]|SFS = [b]|说明匹配此垃圾邮件规则。|
|`IPV=CAL`|邮件已获得垃圾邮件筛选器的允许，因为 IP 地址已在连接筛选器的 IP 允许列表中指定。|
|`H=<EHLOstring>`|连接的电子邮件服务器的 HELO 或 EHLO 字符串。|
|`PTR=<ReverseDNS>`|发送 IP 地址的 PTR 记录，也被称为反向 DNS 地址。|

邮件被过滤为垃圾邮件，如下所示的示例**custom_data**值：

`S:SFA=SUM|SFV=SPM|IPV=CAL|SRV=BULK|SFS=470454002|SFS=349001|SCL=9|SCORE=-1|LIST=0|DI=SN|RD=ftmail.inc.com|H=ftmail.inc.com|CIP=98.129.140.74|SFP=1501|ASF=1|CTRY=US|CLTCTRY=|LANG=en|LAT=287|LAT=260|LAT=18;`

#### <a name="malware-filter-agent"></a>恶意软件筛选器代理

**Custom_data**值开头的`S:AMA`是从恶意软件筛选器代理。下表中描述的关键详细信息：

|**值**|**说明**|
|:-----|:-----|
|AMA = SUM|v=1|` or `AMA=EV|v = 1|邮件已确定包含恶意软件。`SUM`指示恶意软件可能已被检测到任意数量的引擎。`EV`表示特定引擎检测到恶意软件。当恶意软件检测到某个引擎这会触发后续操作。|
|`Action=r`|邮件已被替换。|
|`Action=p`|邮件已被忽略。|
|`Action=d`|邮件已被延迟。|
|`Action=s`|邮件已删除。|
|`Action=st`|邮件已被忽略。|
|`Action=sy`|邮件已被忽略。|
|`Action=ni`|邮件已被拒绝。|
|`Action=ne`|邮件已被拒绝。|
|`Action=b`|邮件已被阻止。|
|`Name=<malware>`|已检测到的恶意软件的名称。|
|`File=<filename>`|恶意软件中包含的文件名称。|

为包含恶意软件的邮件示例**custom_data**值如下所示：

`S:AMA=SUM|v=1|action=b|error=|atch=1;S:AMA=EV|engine=M|v=1|sig=1.155.974.0|name=DOS/Test_File|file=filename;S:AMA=EV|engine=A|v=1|sig=201707282038|name=Test_File|file=filename`

#### <a name="transport-rule-agent"></a>传输规则代理

**Custom_data**值开头的`S:TRA`是传输规则代理的邮件流规则 （也称作传输规则）。下表中描述的关键详细信息：

|**值**|**说明**|
|:-----|:-----|
|ETR|ruleId =<guid>`|匹配的规则 ID。|
|`St=<datetime>`|日期和时间发生规则匹配时的 UTC。|
|`Action=<ActionDefinition>`|应用操作。有关可用操作的列表，请参阅[Mail flow 规则操作在 Exchange Online](https://technet.microsoft.com/library/jj919237.aspx)。|
|`Mode=<Mode>`|规则的模式。有效值为：<br/>•**强制**： 将强制实施规则的所有操作。 <br/>•**使用策略提示测试：**： 将发送所有策略提示操作，但其他强制实施操作不会作用于。 <br/>•**不使用策略提示的情况下测试**： 日志文件中将列出操作，但不是会以任何方式通知发件人和强制实施操作不会作用于。|

示例**custom_data**值的邮件的邮件流规则的条件相匹配如下所示：

`S:TRA=ETR|ruleId=19a25eb2-3e43-4896-ad9e-47b6c359779d|st=7/17/2017 12:31:25 AM|action=ApplyHtmlDisclaimer|sev=1|mode=Enforce`
