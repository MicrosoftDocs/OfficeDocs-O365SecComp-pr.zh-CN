# <a name="create-a-legal-hold-notice"></a>创建合法保留通告

使用高级电子数据展示 （预览） custodian communications，组织可以管理其周围与管理员的工作流。通过通信工具，法律团队还可以系统地发送、 收集和跟踪合法保留通知。灵活的创建过程还允许团队自定义保留通知工作流和中向管理员发送通知的内容。 

文章概述了保留通知工作流中的步骤。

## <a name="step-1-specify-communication-details"></a>步骤 1： 指定通信详细信息

第一步是指定的合法保留通知适当的详细信息或其他 custodian 通信。 

1. 在安全 & 合规性中心中，转到**电子数据展示 > 高级电子数据展示 （预览）** 以便在组织中显示的情况下的列表。
   
2. 单击**Communications**选项卡，然后单击**新的通信**。
   
3. 在**名称的通信**页中，指定以下 （必需） 的通信的详细信息。

    - **名称**： 这是进行通信的名称。
    
    - **颁发部主管**： 下拉列表中显示的案例成员列表。将代表指定颁发官发送每向管理员发送通知。

4. 单击"下一步"。

## <a name="step-2-define-the-portal-content"></a>步骤 2： 定义门户网站内容

接下来，您可以创建并添加保留通知的内容。在**定义门户网站内容**页上的**创建通信**向导中，指定保留通知的内容。此内容将自动追加到颁发、 重新问题、 提醒和升级通知。此外，此内容将出现在 custodian 的合规性门户。 

创建门户网站内容：

1. 类型 （或剪切和从另一个文档 paster） 您保留注意在文本框中的门户网站内容。 

2. 您的通知，以自定义通知和共享 Custodian 合规性门户中插入合并变量。

3. 单击"下一步"。

  >[!Tip]
  >若要了解更多有关如何可以自定义的内容和门户网站内容的格式，请参阅[使用 Communications 编辑器](using-communications-editor.md)。

## <a name="step-3-set-the-required-notifications"></a>步骤 3： 设置所需的通知

您已定义的保留标记内容后，您可以设置围绕发送和管理通知过程的工作流。通知要发送通知的电子邮件及诊后跟踪与管理员。添加到通信的每个 custodian 将收到相同的通知。 

若要设置和发送保持状态通知，必须包括颁发，重新发放并释放通知。

### <a name="issuance-notification"></a>颁发通知 

创建通信后，由指定的发出官启动**颁发通知**。颁发通知是首次通讯发送到 custodian 以告知他们有关其保留义务。 

创建发布通知：

1. 在**颁发**图块，单击**编辑**。
   
2. 如有必要，将其他案例成员或人员添加到**抄送**和**密件抄送**字段。若要向这些字段添加多个用户，请用分号分隔电子邮件地址。
   
3. 指定的 （必需） 的通知的**主题**。
   
4. 指定的内容或您希望向 （必需） 管理员提供的其他说明。请注意，您在步骤 2 中定义的门户内容会添加到颁发通知的末尾。 
   
5. 单击“保存”**** 

### <a name="re-issuance-notification"></a>重新发放通知 

随着案例，管理员可能需要比先前已经指示保留其他或更少的数据。更新的保留标记内容后，重新发放通知提醒有关更改其保留义务的管理员。

要创建重新发放通知： 

1. 在**重新发出**图块中，单击**编辑**。
   
2. 如有必要，将其他案例成员或人员添加到**抄送**和**密件抄送**字段。若要向这些字段添加多个用户，请用分号分隔电子邮件地址。
   
3. 指定的 （必需） 的通知的**主题**。
   
4. 指定的内容或您希望向 （必需） 管理员提供的其他说明。请注意，您在步骤 2 中定义的门户内容会添加到末尾的重新发放通知。
   
5. 单击“**保存**”。

>[!Note]
>如果修改保留通知时，将自动将重新发放通知发送给所有分配给通知的管理员。发送通知后，管理员将需要重新确认其保留标记。如果您已将任何提醒或升级的工作流设置，这些将还重新启动。 

### <a name="release-notification"></a>释放通知

解决与否后或管理员受制于不再是保留内容，您可以释放从案例 custodian。如果 custodian 以前颁发保留通知，释放通知可用于通知管理员他们具有其义务从已释放。

要创建发布通知： 

1. 在**版本**图块，单击**编辑**。
   
2. 如有必要，将其他案例成员或人员添加到**抄送**和**密件抄送**字段。若要向这些字段添加多个用户，请用分号分隔电子邮件地址。
   
3. 指定的 （必需） 的通知的**主题**。
   
4. 指定的内容或您希望向 （必需） 管理员提供的其他说明。
   
5. 单击**保存**，然后转到下一步。 

## <a name="optional-step-4-set-the-optional-notifications"></a>（可选）步骤 4： 设置可选的通知

（可选） 您可以简化关注与未响应管理员创建和计划自动的提醒和升级通知的工作流。

### <a name="reminders"></a>提醒

已发送保持状态通知后，您可以进行后续工作与未响应管理员通过定义提醒工作流。 

计划提醒：

1. 在**提醒**图块，单击**编辑**。
   
2. 通过启用 （必需） 的**状态**切换启用**提醒**工作流。
   
3. 指定**提醒时间间隔 （以天为单位）** （必需）。这是发送第一个和后续提醒通知前要等待的天数。例如，如果将提醒时间间隔设置为 7 天，然后第一个提醒将发送 7 天后最初颁发保留通知。此外将发送所有后续提醒每 7 天。
   
4. 指定**的提醒数**（必需）。此字段指定多少提醒，以便将发送到未响应的管理员。例如，如果将提醒数设置为 3，管理员会收到 3 提醒的最大值。管理员确认保留通知后，不再将向用户发送提醒。
   
5. 指定的 （必需） 的通知的**主题**。 
   
6. 指定的内容或您希望向 （必需） 管理员提供的其他说明。请注意，您在步骤 2 中定义的门户内容会添加到提醒通知的末尾。
   
7. 单击**保存**，然后转下一步。

### <a name="escalations"></a>升级 

在某些情况下，您可能需要跟踪的其他方式与未响应的管理员。如果管理员不会确认保留通知后接收指定的数量的提醒，法律团队可以指定要自动将升级通知发送给 custodian 和经理的工作流。

计划升级：

1. 在**升级**图块，单击**编辑**。
   
2. 通过启用的**状态**切换启用**升级**工作流。
   
3. 指定**升级时间间隔 （以天为单位）** （必需）。 
   
4. 指定**升级数**（必需）。此字段指定多少升级将发送到未响应的管理员。例如，如果将升级数设置为 3，然后升级通知会发送给 custodian 和经理 3 次，最多。管理员确认保留通知后，将无法再发送升级。 
   
5. 指定的 （必需） 的通知的**主题**。 
   
6. 指定的内容或您希望向 （必需） 管理员提供的其他说明。请注意，您在步骤 2 中定义的门户内容会添加到末尾的升级通知。
   
7. 单击**保存**，然后转下一步。
   
## <a name="step-5-assign-custodians"></a>步骤 5： 分配管理员 

尚未最终通知的内容后，请选择管理员向其发送通知。 

若要添加管理员：

1. 将管理员分配给通信中，单击其姓名旁的复选框。

    创建通信后，通知工作流将自动应用于所选的管理员。
   
2. 单击**下一步**检查的通信设置和详细信息。
 
>[!NOTE]
>只能添加管理员已添加到这种情况，且尚未发送其他通知在这种情况。

## <a name="step-6-review-settings"></a>步骤 6： 查看设置

查看设置，然后单击**发送**来完成通信后，系统将自动启动通信工作流通过发送颁发通知。
