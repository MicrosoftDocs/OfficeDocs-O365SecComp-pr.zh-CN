# <a name="create-a-legal-hold-notice"></a>创建合法保留通知

通过使用高级电子数据展示 (预览版) 保管人通信, 组织可以管理其与保管人通信的工作流。通过通信工具, 法律团队可以系统地发送、收集和跟踪合法保留通知。灵活的创建过程还允许团队自定义保留通知工作流以及发送给保管人的通知中的内容。 

本文概述了保留通知工作流中的步骤。

## <a name="step-1-specify-communication-details"></a>步骤 1: 指定通信详细信息

第一步是为合法保留通知或其他保管人通信指定相应的详细信息。 

1. 在安全 & 合规性中心中, 转到**电子数据展示 > 高级电子数据展示 (预览)** 以显示您的组织中的案例列表。
   
2. 单击 "**通信**" 选项卡, 然后单击 "**新建通信**"。
   
3. 在 "**名称通信**" 页上, 指定以下 (必需) 的通信详细信息。

    - **名称**: 这是通信的名称。
    
    - **签发专员**: 下拉列表显示事例成员的列表。发送给保管人的每个通知将代表指定的颁发官发送。

4. 单击“下一步”。****

## <a name="step-2-define-the-portal-content"></a>步骤 2: 定义门户内容

接下来, 您可以创建和添加保留通知的内容。在 "**创建通信**向导" 中的 "**定义门户内容**" 页上, 指定保留通知的内容。此内容将自动追加到发布、重新发布、提醒和上报通知中。此外, 此内容将显示在保管人的合规性门户中。 

若要创建门户内容, 请执行以下操作:

1. 在门户内容的 textbox 中键入 (或从另一个文档中剪切和 paster) 您的保留通知。 

2. 将合并变量插入到通知中, 以自定义通知并共享保管人合规性门户。

3. 单击“下一步”。****

  >[!Tip]
  >若要了解有关如何自定义门户内容的内容和格式的详细信息, 请参阅[使用通信编辑器](using-communications-editor.md)。

## <a name="step-3-set-the-required-notifications"></a>步骤 3: 设置所需的通知

定义保留通知的内容后, 可以在发送和管理通知过程周围设置工作流。通知是发送给保管人以通知和跟进的电子邮件。每个添加到通信的保管人都会收到相同的通知。 

若要设置和发送保留通知, 必须包括发布通知、重新颁发通知和发布通知。

### <a name="issuance-notification"></a>颁发通知 

在创建通信之后, 将由指定的颁发官启动**颁发通知**。颁发通知是发送给管理员的第一次通信, 告知他们其保留义务。 

创建发布通知的步骤:

1. 在**发布**磁贴中, 单击 "**编辑**"。
   
2. 如有必要, 向 **"抄送**" 和 **"密件抄送**" 字段中添加其他事例成员或员工。若要向这些字段中添加多个用户, 请使用分号分隔电子邮件地址。
   
3. 指定通知的**主题**(必需)。
   
4. 指定您希望为保管人提供的内容或其他说明 (必需)。请注意, 在步骤2中定义的门户内容将添加到发布通知的末尾。 
   
5. 单击“保存”**** 

### <a name="re-issuance-notification"></a>重新发布通知 

在这种情况下, 可能需要保管人来保留比以前所指示的更多或少的数据。更新保留通知的内容后, 重新发布通知将向保管人通知保管人有关其保留义务的更改。

创建重新发布通知的步骤: 

1. 在重新**颁发**磁贴中, 单击 "**编辑**"。
   
2. 如有必要, 向 **"抄送**" 和 **"密件抄送**" 字段中添加其他事例成员或员工。若要向这些字段中添加多个用户, 请使用分号分隔电子邮件地址。
   
3. 指定通知的**主题**(必需)。
   
4. 指定您希望为保管人提供的内容或其他说明 (必需)。请注意, 在步骤2中定义的门户内容将添加到重新发布通知的末尾。
   
5. 单击“**保存**”。

>[!Note]
>如果修改了保留通知, 则将自动向分配给此通知的所有保管人发送重新发布通知。发送通知后, 将要求保管人重新确认其保留通知。如果您已设置任何提醒或升级工作流, 这些工作流也将重新启动。 

### <a name="release-notification"></a>发布通知

在解决问题或管理员不再受保留内容的限制后, 可以从案例中释放该保管人。如果保管人先前已发出保留通知, 则可以使用发布通知来提醒保管人他们已从其义务中发布。

创建发布通知: 

1. 在 "**发布**" 磁贴中, 单击 "**编辑**"。
   
2. 如有必要, 向 **"抄送**" 和 **"密件抄送**" 字段中添加其他事例成员或员工。若要向这些字段中添加多个用户, 请使用分号分隔电子邮件地址。
   
3. 指定通知的**主题**(必需)。
   
4. 指定您希望为保管人提供的内容或其他说明 (必需)。
   
5. 单击 "**保存**", 然后转到下一步。 

## <a name="optional-step-4-set-the-optional-notifications"></a>Optional步骤 4: 设置可选通知

(可选) 您可以通过创建和安排自动提醒和上报通知来简化与未响应的保管人进行后续工作的工作流。

### <a name="reminders"></a>提醒

发送保留通知后, 通过定义提醒工作流, 可以跟进不响应保管人的人。 

计划提醒:

1. 在**提醒**磁贴中, 单击 "**编辑**"。
   
2. 通过打开 "**状态**" 切换 (必需) 启用**提醒**工作流。
   
3. 指定**提醒间隔 (以天**为单位) (必需)。这是发送第一个和跟进提醒通知之前要等待的天数。例如, 如果将提醒间隔设置为7天, 则在最初发出保留通知后, 将在7天内发送第一个提醒。所有后续提醒也每7天发送一次。
   
4. 指定**提醒数**(必需)。此字段指定要发送给未响应的保管人的提醒数。例如, 如果将提醒数设置为 3, 则管理员最多可收到3个提醒。在保管人确认保留通知后, 将不再向该用户发送提醒。
   
5. 指定通知的**主题**(必需)。 
   
6. 指定您希望为保管人提供的内容或其他说明 (必需)。请注意, 您在步骤2中定义的门户内容将添加到提醒通知的末尾。
   
7. 单击 "**保存**", 然后转到下一步。

### <a name="escalations"></a>上报 

在某些情况下, 您可能需要使用其他方法来跟踪不响应保管人的用户。如果管理员在收到指定数量的提醒后未确认保留通知, 则法律团队可以指定一个工作流, 以自动向保管人和他们的经理发送上报通知。

计划升级:

1. 在**升级**图块中, 单击 "**编辑**"。
   
2. 通过打开 "**状态**" 切换来启用**升级**工作流。
   
3. 指定**升级间隔 (以天**为单位) (必需)。 
   
4. 指定**升级次数**(必需)。此字段指定要发送给未响应的保管人的升级次数。例如, 如果将呈报次数设置为 3, 则会将呈报通知发送给保管人者及其经理, 最多为3次。在保管人确认保留通知后, 将不再发送升级。 
   
5. 指定通知的**主题**(必需)。 
   
6. 指定您希望为保管人提供的内容或其他说明 (必需)。请注意, 在步骤2中定义的门户内容将添加到上报通知的末尾。
   
7. 单击 "**保存**", 然后转到下一步。
   
## <a name="step-5-assign-custodians"></a>步骤 5: 分配保管人 

完成通知的内容后, 选择要将通知发送到的保管人。 

若要添加保管人:

1. 通过单击其名称旁边的复选框为保管人分配通信。

    在创建通信后, 通知工作流将自动应用于所选的保管人。
   
2. 单击 "**下一步**" 查看通信设置和详细信息。
 
>[!NOTE]
>您只能添加已添加到此案例中的保管人, 并且在该案例中还未发送其他通知。

## <a name="step-6-review-settings"></a>步骤 6: 查看设置

查看设置并单击 "**发送**" 以完成通信后, 系统将通过发送颁发通知自动启动通信工作流。
